# transcription.db を開く方法

## 推奨アプリケーション

### 1. DB Browser for SQLite（推奨・無料）
**最も簡単で使いやすいSQLiteビューアー**

**インストール方法:**
```bash
# Homebrewを使用
brew install --cask db-browser-for-sqlite
```

**または公式サイトから:**
- https://sqlitebrowser.org/
- ダウンロードしてアプリケーションフォルダに配置

**使い方:**
1. DB Browser for SQLiteを起動
2. 「データベースを開く」をクリック
3. `/Users/kayoko.namba/Desktop/transcription/transcription.db` を選択
4. 「Browse Data」タブでテーブルを閲覧

**特徴:**
- ✅ 無料・オープンソース
- ✅ 直感的なUI
- ✅ テーブル構造の確認が簡単
- ✅ SQLクエリの実行が可能
- ✅ データの編集が可能

### 5. コマンドライン（ターミナル）
**軽量で高速、スクリプト化可能**

**基本的な使い方:**
```bash
cd /Users/kayoko.namba/Desktop/transcription

# データベースを開く（対話モード）
sqlite3 transcription.db

# SQLコマンドの例
.tables                    # テーブル一覧
.schema files              # filesテーブルの構造
SELECT * FROM files;       # filesテーブルの全データ
SELECT * FROM segments LIMIT 10;  # segmentsテーブルの最初の10件
.quit                      # 終了
```

**便利なコマンド:**
```bash
# テーブル一覧とレコード数を表示
sqlite3 transcription.db ".tables" ".mode column" "SELECT 'files' as table_name, COUNT(*) as count FROM files UNION ALL SELECT 'segments', COUNT(*) FROM segments;"

# CSV形式でエクスポート
sqlite3 transcription.db "SELECT * FROM files;" > files.csv

# 整形して表示
sqlite3 transcription.db -header -column "SELECT * FROM files;"
```

## データベースの内容

### files テーブル
- 処理された音声ファイルの情報
- ファイル名、パス、サイズ、長さ、処理状態など

### segments テーブル
- 文字起こしのセグメント情報
- 話者、テキスト、開始時間、終了時間など

---

## 便利なSQLクエリ例

### ファイル一覧を表示
```sql
SELECT id, filename, status, duration, processed_at 
FROM files 
ORDER BY processed_at DESC;
```

### セグメントを表示（特定ファイル）
```sql
SELECT s.segment_index, s.speaker, s.text, s.start_time, s.end_time
FROM segments s
JOIN files f ON s.file_id = f.id
WHERE f.filename = '輪読会10.mp4'
ORDER BY s.segment_index;
```

### 話者別のセグメント数
```sql
SELECT speaker, COUNT(*) as count
FROM segments
WHERE speaker IS NOT NULL
GROUP BY speaker
ORDER BY count DESC;
```

### 統計情報
```sql
SELECT 
    (SELECT COUNT(*) FROM files) as total_files,
    (SELECT COUNT(*) FROM files WHERE status = 'completed') as completed_files,
    (SELECT COUNT(*) FROM segments) as total_segments,
    (SELECT SUM(duration) FROM files WHERE duration IS NOT NULL) as total_duration;
```

## トラブルシューティング

### データベースがロックされている
- 他のプロセス（バッチ処理など）がデータベースを使用している可能性があります
- バッチ処理が完了するまで待つか、処理を停止してください

### ファイルが見つからない
- パスを確認してください: `/Users/kayoko.namba/Desktop/transcription/transcription.db`
- ファイルが存在するか確認: `ls -lh transcription.db`

### 権限エラー
- ファイルの読み取り権限を確認してください
- 必要に応じて: `chmod 644 transcription.db`
