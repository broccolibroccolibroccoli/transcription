# 輪読会ファイルのバッチ処理実行手順

## 概要
複数の輪読会音声ファイルを自動的に処理し、SQLiteデータベースに保存する手順です。

## 前提条件
- Python 3.12以上がインストールされていること
- 仮想環境 `/Users/kayoko.namba/venvs/whisperx` が作成されていること
- 必要なPythonパッケージがインストールされていること
  - `whisperx`
  - `torch`
  - `numpy`
  - `sqlite3` (標準ライブラリ)

## ファイル構成

```
transcription/
├── database_schema.py      # データベーススキーマ定義
├── batch_process.py        # バッチ処理スクリプト
├── db_query.py            # データベース検索・確認用ユーティリティ
├── transcription.db       # SQLiteデータベース（自動生成）
└── 輪読会*.mp4            # 処理対象の音声ファイル
```

## 実行手順

### ステップ1: 仮想環境のアクティベート
```bash
source /Users/kayoko.namba/venvs/whisperx/bin/activate
```

**確認方法:**
```bash
which python
# 出力例: /Users/kayoko.namba/venvs/whisperx/bin/python
```

### ステップ2: プロジェクトディレクトリに移動
```bash
cd /Users/kayoko.namba/Desktop/transcription
```

### ステップ3: データベーススキーマの作成（初回のみ）
```bash
python database_schema.py
```

**出力例:**
```
✅ データベーススキーマを作成しました: transcription.db
```

### ステップ4: バッチ処理の実行
```bash
python batch_process.py
```

**処理内容:**
1. 輪読会ファイルの自動検出（`輪読会*.mp4`, `輪読会*.wav`など）
2. 各ファイルに対して以下を実行:
   - WhisperXによる文字起こし
   - タイムスタンプの補正（Alignment）
   - 話者分離（Diarization）
   - データベースへの保存

**実行例:**
```

============================================================

  1. 輪読会#10.wav
  2. 輪読会#11.mp4
  3. 輪読会#9.mp4
  4. 輪読会10.mp4

[1/4] 処理中...
============================================================
処理開始: 輪読会#10.wav
============================================================
--- ステップ1: WhisperXで文字起こし開始 ---
...
```

### ステップ5: 処理結果の確認

#### 5-1. ファイル一覧の確認
```bash
python db_query.py list
```

#### 5-2. 統計情報の確認
```bash
python db_query.py stats
```

#### 5-3. セグメントの確認

#### 5-4. テキストファイルへのエクスポート
```bash
# 全ファイルをエクスポート
python db_query.py export

# 特定ファイルをエクスポート
python db_query.py export "輪読会10.mp4" output.txt
```

## データベーススキーマ

### files テーブル
| カラム名 | 型 | 説明 |
|---------|-----|------|
| id | INTEGER | 主キー |
| filename | TEXT | ファイル名 |
| filepath | TEXT | ファイルパス |
| file_size | INTEGER | ファイルサイズ（バイト） |
| duration | REAL | 音声の長さ（秒） |
| processed_at | TIMESTAMP | 処理日時 |
| status | TEXT | 処理状態（pending/completed/error） |
| error_message | TEXT | エラーメッセージ（エラー時） |
| created_at | TIMESTAMP | 作成日時 |
| updated_at | TIMESTAMP | 更新日時 |

### segments テーブル
| カラム名 | 型 | 説明 |
|---------|-----|------|
| id | INTEGER | 主キー |
| file_id | INTEGER | ファイルID（外部キー） |
| segment_index | INTEGER | セグメント番号 |
| speaker | TEXT | 話者ID（SPEAKER_00など） |
| text | TEXT | 文字起こしテキスト |
| start_time | REAL | 開始時間（秒） |
| end_time | REAL | 終了時間（秒） |
| created_at | TIMESTAMP | 作成日時 |

## トラブルシューティング

### エラー: データベースがロックされている
- 他のプロセスがデータベースを使用していないか確認してください
- データベースファイルの権限を確認してください

### エラー: ファイルが見つからない
- ファイルパスが正しいか確認してください
- ファイル名のパターンが正しいか確認してください（`輪読会*`）

### エラー: メモリ不足
- `batch_size`を小さくする（`batch_process.py`の12行目）
- 一度に処理するファイル数を減らす

### エラー: Hugging Faceトークンが無効
- `batch_process.py`の48行目のトークンを確認してください
- 環境変数に設定することを推奨します

## 再処理について

同じファイルを再度処理する場合：
- 既存のセグメントは削除され、新しい結果で上書きされます
- ファイル情報は更新されますが、ファイルIDは維持されます

## 注意事項

1. **処理時間**: CPUモードで実行されるため、長時間の音声ファイルは処理に時間がかかります
2. **メモリ使用量**: large-v3モデルはメモリを多く使用します
3. **データベースサイズ**: 大量のファイルを処理するとデータベースサイズが大きくなります
4. **トークンのセキュリティ**: Hugging Faceトークンは環境変数に設定することを推奨します
