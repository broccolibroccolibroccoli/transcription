# 技術構成

## アーキテクチャ概要

```
┌─────────────────────────────────────────────────────────────────┐
│                     Web フロントエンド                            │
│  (Streamlit / Gradio)                                            │
│  - ファイルアップロード                                           │
│  - 処理状況表示                                                   │
│  - 文字起こし・要約結果の表示                                      │
└──────────────────────────┬──────────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────────────┐
│                     Python バックエンド                           │
│  - アップロード受付                                               │
│  - WhisperX 呼び出し（既存 process_audio_file を再利用）          │
│  - 要約処理（LLM API または ローカルモデル）                        │
└──────────────────────────┬──────────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────────────┐
│                     SQLite データベース                           │
│  - files テーブル（ファイル情報）                                  │
│  - segments テーブル（文字起こしセグメント）                        │
│  - summaries テーブル（要約結果）※新規追加                         │
└─────────────────────────────────────────────────────────────────┘
```

## 技術スタック

| レイヤー | 技術 | 備考 |
|---------|------|------|
| フロントエンド | **Streamlit** または **Gradio** | シンプルなUI、Pythonのみで完結 |
| 文字起こし | WhisperX | 既存実装を再利用 |
| 要約 | OpenAI API / Anthropic API / ローカルLLM | 選択可能 |
| DB | SQLite | 既存スキーマを拡張 |
| 実行環境 | Python 3.12 + venv | 既存 venv を利用 |

## 推奨: Streamlit を選ぶ理由

- 既存のPythonコードとの統合が容易
- ファイルアップロードコンポーネントが標準搭載
- デプロイが簡単（`streamlit run app.py`）
- 日本語対応が良好

## 要約機能の実装オプション

| オプション | メリット | デメリット |
|-----------|---------|-----------|
| **OpenAI API** | 高品質、実装が簡単 | 課金発生、APIキー必要 |
| **Anthropic API** | 日本語に強い | 同上 |
| **ローカルLLM** (llama.cpp等) | 無料、オフライン可 | 環境構築が重い |
| **ルールベース要約** | 無料、依存なし | 品質が限定的 |

初期実装では **OpenAI API** または **環境変数で切り替え可能** にすることを推奨。
