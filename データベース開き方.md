# transcription.db を開く方法

## 方法1: 自動起動スクリプトを使用（推奨・最も簡単）

### 実行方法
```bash
cd /Users/kayoko.namba/Desktop/transcription
./open_db_v2.sh
```

このスクリプトは自動的に最適な方法を選択します：
1. DB Browser for SQLiteがインストールされている場合は自動起動
2. 仮想環境のPythonでGUIアプリを起動
3. コマンドライン版（sqlite3）を使用
4. Python CLIビューアーを使用

---

## 方法2: コマンドライン版ビューアー（tkinter不要）

### 対話モード
```bash
cd /Users/kayoko.namba/Desktop/transcription
python3 open_db_cli.py
```

### クイックコマンド
```bash
# テーブル一覧
python3 open_db_cli.py tables

# 統計情報
python3 open_db_cli.py stats

# テーブルデータを表示
python3 open_db_cli.py show files
python3 open_db_cli.py show segments 50  # 50件表示
```

### 機能
- ✅ テーブル一覧の表示
- ✅ テーブルデータの閲覧
- ✅ SQLクエリの実行
- ✅ 統計情報の表示
- ✅ tkinter不要（どの環境でも動作）

## 方法3: コマンドラインで開く

### 基本的な使い方
```bash
cd /Users/kayoko.namba/Desktop/transcription
sqlite3 transcription.db
```

### 便利なコマンド
```sql
-- テーブル一覧
.tables

-- filesテーブルの構造
.schema files

-- filesテーブルの全データ
SELECT * FROM files;

-- segmentsテーブルの最初の10件
SELECT * FROM segments LIMIT 10;

-- 統計情報
SELECT 
    (SELECT COUNT(*) FROM files) as total_files,
    (SELECT COUNT(*) FROM segments) as total_segments;

-- 終了
.quit
```

### ワンライナーで確認
```bash
# ファイル一覧を表示
sqlite3 transcription.db -header -column "SELECT id, filename, status, duration FROM files;"

# セグメント数を表示
sqlite3 transcription.db "SELECT COUNT(*) as segment_count FROM segments;"

# 統計情報を表示
sqlite3 transcription.db -header -column "
SELECT 
    (SELECT COUNT(*) FROM files) as total_files,
    (SELECT COUNT(*) FROM files WHERE status = 'completed') as completed_files,
    (SELECT COUNT(*) FROM segments) as total_segments;
"
```

## トラブルシューティング

### GUIアプリケーションが起動しない場合

**Python3がインストールされているか確認:**
```bash
python3 --version
```

**必要なライブラリがインストールされているか確認:**
```bash
python3 -c "import tkinter; print('OK')"
```

**macOSでtkinterが使えない場合:**
```bash
# Python3を再インストール（Homebrew経由）
brew install python3
```

### データベースがロックされている場合

他のプロセス（バッチ処理など）がデータベースを使用している可能性があります。

**確認方法:**
```bash
lsof transcription.db
```

**対処法:**
- バッチ処理が完了するまで待つ
- または処理を停止する

### ファイルが見つからない場合

**パスを確認:**
```bash
ls -lh /Users/kayoko.namba/Desktop/transcription/transcription.db
```

**権限を確認:**
```bash
chmod 644 transcription.db
```
