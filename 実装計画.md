# 実装計画

## フェーズ分け

### フェーズ1: 基盤整備（1〜2日）

| タスク | 内容 |
|--------|------|
| DBスキーマ拡張 | `summaries` テーブル追加 |
| 処理モジュールの共通化 | `process_audio_file` をファイルパターンに依存しない形にリファクタ |
| アップロード用ディレクトリ | `uploads/` を作成し、アップロードファイルを保存 |

### フェーズ2: Web UI（2〜3日）

| タスク | 内容 |
|--------|------|
| Streamlit 導入 | `pip install streamlit` |
| アップロード画面 | `st.file_uploader` でファイル受付 |
| 処理実行 | アップロード → 保存 → `process_audio_file` 呼び出し |
| 結果表示 | セグメント一覧を表示 |

### フェーズ3: 要約機能（1〜2日）

| タスク | 内容 |
|--------|------|
| 要約API連携 | OpenAI / Anthropic クライアント実装 |
| 要約プロンプト設計 | インタビュー向けの要約プロンプト |
| 要約保存 | `summaries` テーブルに保存 |
| 要約表示 | 詳細画面に要約セクション追加 |

### フェーズ4: 仕上げ（1日）

| タスク | 内容 |
|--------|------|
| エクスポート | 文字起こし＋要約をまとめてダウンロード |
| エラーハンドリング | 処理失敗時のメッセージ表示 |
| ドキュメント | アプリの起動方法・使い方 |

## ファイル構成（実装後）

```
transcription/
├── app.py                 # Streamlit メインアプリ（新規）
├── summarizer.py          # 要約処理モジュール（新規）
├── uploads/               # アップロードファイル保存先（新規）
├── batch_process.py       # 既存（process_audio_file を共通化）
├── database_schema.py     # 既存（summaries テーブル追加）
├── db_query.py            # 既存（要約取得メソッド追加）
├── transcription.db       # 既存
├── docs/                  # 設計ドキュメント
│   ├── アプリ設計案_概要.md
│   ├── 技術構成.md
│   ├── 画面・機能仕様.md
│   └── 実装計画.md
└── requirements_app.txt   # アプリ用依存関係（新規）
```

## 必要な追加パッケージ

```
streamlit>=1.28.0
openai>=1.0.0          # 要約にOpenAI APIを使う場合
# または
anthropic>=0.18.0      # 要約にAnthropic APIを使う場合
```

## 起動コマンド（想定）

```bash
source /Users/kayoko.namba/venvs/whisperx/bin/activate
cd /Users/kayoko.namba/Desktop/transcription
pip install streamlit openai   # 初回のみ
streamlit run app.py
```

ブラウザで `http://localhost:8501` にアクセス。
