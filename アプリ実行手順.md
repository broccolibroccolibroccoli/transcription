# インタビュー音声 文字起こし・要約アプリ 実行手順

## 概要

インタビューなどの音声ファイル（mp4, wav, mp3, m4a, flac）をブラウザからアップロードするか、公開されているYouTube動画のURLを入力し、文字起こし・話者分離を行うWebアプリの実行手順です。

## 前提条件

- Python 3.12以上がインストールされていること
- 仮想環境 `/Users/kayoko.namba/venvs/whisperx` が作成されていること
- 必要なPythonパッケージがインストールされていること
  - `whisperx`
  - `torch`
  - `numpy`
  - `streamlit`
  - `openai`（要約機能を使用する場合）
  - `yt-dlp`（YouTube URL対応に必要）
- FFmpeg がシステムにインストールされていること（YouTube音声ダウンロードに必要）

## ファイル構成

```
transcription/
├── .streamlit/
│   └── config.toml        # KARTEテーマ設定
├── app.py                 # Streamlit メインアプリ
├── batch_process.py       # 文字起こし・話者分離処理
├── youtube_utils.py       # YouTube音声ダウンロード（公開動画対応）
├── database_schema.py     # データベーススキーマ定義
├── db_query.py            # データベース検索・確認用ユーティリティ
├── transcription.db      # SQLiteデータベース（自動生成）
├── uploads/               # アップロードファイル保存先（自動生成）
├── requirements_app.txt   # アプリ用追加パッケージ
└── アプリ実行手順.md       # 本ドキュメント
```

## 実行手順

### ステップ1: 仮想環境のアクティベート

```bash
source /Users/kayoko.namba/venvs/whisperx/bin/activate
```

**確認方法:**
```bash
which python
# 出力例: /Users/kayoko.namba/venvs/whisperx/bin/python
```

### ステップ2: プロジェクトディレクトリに移動

```bash
cd /Users/kayoko.namba/Desktop/transcription
```

### ステップ3: 追加パッケージのインストール（初回のみ）

```bash
pip install -r requirements_app.txt
```

または個別にインストールする場合:

```bash
pip install streamlit openai python-dotenv yt-dlp
```

### ステップ4: 要約機能を使う場合の環境変数設定（任意）

要約を生成するには OpenAI API キーが必要です。

```bash
export OPENAI_API_KEY="sk-..."
```

※ 未設定の場合、文字起こし・話者分離は実行されますが、要約はスキップされます。

### ステップ5: アプリの起動

```bash
streamlit run app.py
```

**出力例:**
```
  You can now view your Streamlit app in your browser.

  Local URL: http://localhost:8501
  Network URL: http://192.168.0.25:8501
```

### ステップ6: ブラウザでアクセス

ブラウザで `http://localhost:8501` を開きます。自動で開かない場合は、上記の Local URL を手動で入力してください。

## アプリの使い方

### 1. ファイルのアップロード

1. サイドバーの「ファイルアップロード」で音声ファイルを選択
2. 対応形式: mp4, wav, mp3, m4a, flac
3. 「文字起こしを開始」ボタンをクリック

### 2. 処理の流れ

処理は以下の順序で自動実行されます：

| ステップ | 処理内容 |
|---------|---------|
| 1 | WhisperXによる文字起こし |
| 2 | タイムスタンプの補正（Alignment） |
| 3 | 話者分離（Diarization） |
| 4 | データベースへの保存 |
| 5 | 要約の生成（OPENAI_API_KEY設定時のみ） |

※ 処理には数分かかる場合があります。

### 3. 結果の確認

1. メイン画面の「登録ファイル一覧」で処理済みファイルを確認
2. ファイル名をクリックして展開
3. 「文字起こしを表示」ボタンをクリック
4. **要約**（設定時）→ **文字起こし**の順で表示されます

## データベーススキーマ

### files テーブル
| カラム名 | 型 | 説明 |
|---------|-----|------|
| id | INTEGER | 主キー |
| filename | TEXT | ファイル名 |
| filepath | TEXT | ファイルパス |
| file_size | INTEGER | ファイルサイズ（バイト） |
| duration | REAL | 音声の長さ（秒） |
| status | TEXT | 処理状態（pending/completed/error） |
| error_message | TEXT | エラーメッセージ（エラー時） |
| created_at | TIMESTAMP | 作成日時 |
| updated_at | TIMESTAMP | 更新日時 |

### segments テーブル
| カラム名 | 型 | 説明 |
|---------|-----|------|
| id | INTEGER | 主キー |
| file_id | INTEGER | ファイルID（外部キー） |
| segment_index | INTEGER | セグメント番号 |
| speaker | TEXT | 話者ID（SPEAKER_00など） |
| text | TEXT | 文字起こしテキスト |
| start_time | REAL | 開始時間（秒） |
| end_time | REAL | 終了時間（秒） |
| created_at | TIMESTAMP | 作成日時 |

### summaries テーブル（要約用）
| カラム名 | 型 | 説明 |
|---------|-----|------|
| id | INTEGER | 主キー |
| file_id | INTEGER | ファイルID（外部キー） |
| summary_type | TEXT | 要約タイプ（'full'） |
| content | TEXT | 要約本文 |
| model_used | TEXT | 使用したLLM（例: gpt-4o-mini） |
| created_at | TIMESTAMP | 作成日時 |

## トラブルシューティング

### エラー: アプリが起動しない

- 仮想環境がアクティベートされているか確認
- `pip install streamlit` で Streamlit がインストールされているか確認

### エラー: 文字起こしが完了しない

- 処理に数分かかることがあります。そのままお待ちください
- 長時間の音声ファイルは特に時間がかかります
- ターミナルに表示されるエラーメッセージを確認してください

### エラー: 要約が表示されない

- `OPENAI_API_KEY` が環境変数に設定されているか確認
- APIキーが有効か確認
- 要約は文字起こし完了後に生成されます。未設定の場合は要約のみスキップされ、文字起こしは正常に完了します

### エラー: データベースがロックされている

- 他のプロセス（別のStreamlit、db_query.py等）がデータベースを使用していないか確認
- アプリを再起動してください

### エラー: メモリ不足

- `batch_process.py` の `batch_size` を小さくする（デフォルト: 4）
- 長時間の音声ファイルは分割してアップロードすることを検討

## アプリの停止

ターミナルで `Ctrl + C` を押してアプリを停止します。

## 再起動手順

```bash
# 既存のStreamlitプロセスを停止
pkill -f "streamlit run" 2>/dev/null

# 仮想環境をアクティベート
source /Users/kayoko.namba/venvs/whisperx/bin/activate

# プロジェクトディレクトリに移動
cd /Users/kayoko.namba/Desktop/transcription

# アプリを起動
streamlit run app.py
```
